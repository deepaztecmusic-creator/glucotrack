Help
Answers to common questions
How do I log a glucose reading?‚ñº
Tap the blue Log Reading button at the bottom of the screen. Enter your glucose value from your meter in mmol/L, select the context (e.g. Before Meal), add any carbs or insulin, then tap Save.
How does the Glucose Forecast work?‚ñº
The forecast uses your recent readings to calculate a trend, then factors in any active carbs and insulin you've logged. It predicts your glucose 1‚Äì3 hours ahead. Log regularly for best accuracy.
What is Time In Range (TIR)?‚ñº
Time In Range is the percentage of your readings that fall within your target glucose range (default 3.9‚Äì10.0 mmol/L). The ADA recommends a TIR of at least 70% for most people with diabetes.
How is estimated A1c calculated?‚ñº
Estimated A1c is calculated using the formula: A1c ‚âà (average mmol/L √ó 1.5927 + 1.628). This is an estimate only ‚Äî always get a lab HbA1c test from your doctor for accurate results.
How do I scan a meal with AI?‚ñº
Tap üì∑ Scan Meal in the top bar. Point your camera at your meal and tap capture. The AI will identify the food and estimate carbohydrate content. You can then log it and get an insulin suggestion based on your carb ratio.
How do I export my data?‚ñº
Go to Settings ‚Üí Data ‚Üí Export All Readings (CSV). This downloads a spreadsheet with all your logged readings, which you can share with your healthcare team.
Is my data private?‚ñº
Yes. All your glucose readings and meal data are stored locally on your device only. Nothing is uploaded to any server. The only external connection is the AI meal analysis feature, which sends meal photos to Claude AI for analysis.
Need more help? Contact your diabetes care team or endocrinologist. GlucoTrack is not a medical device and does not replace professional medical advice.

‚ö° One-Time Setup Required

Abbott's API blocks direct browser connections. You need to deploy a free personal proxy using Cloudflare Workers (free forever, takes ~2 minutes).

Step 1 ‚Äî Create your Worker
  1. Go to workers.cloudflare.com ‚Üí sign up free
  2. Click Create Application ‚Üí Create Worker
  3. Replace all the default code with the code below
  4. Click Save and Deploy
  5. Copy your Worker URL (looks like https://xxx.workers.dev)
Worker Code ‚Äî copy and paste this:
export default {
  async fetch(request) {
    if (request.method === 'OPTIONS') {
      return new Response(null, { headers: {
        'Access-Control-Allow-Origin': '*',
        'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
        'Access-Control-Allow-Headers': '*',
      }});
    }
    const url = new URL(request.url);
    const target = url.searchParams.get('url');
    if (!target) return new Response('Missing ?url=', { status: 400 });
    const resp = await fetch(new Request(target, {
      method: request.method,
      headers: request.headers,
      body: request.method !== 'GET' ? request.body : undefined,
    }));
    return new Response(resp.body, { status: resp.status, headers: {
      'Content-Type': 'application/json',
      'Access-Control-Allow-Origin': '*',
    }});
  }
}
